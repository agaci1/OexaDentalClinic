Actor:
Admin

 User Scenarios

1. Manage Users  
Admin views the list of users  
Admin can add, edit, or delete users  

2. View Reports  
Admin accesses system reports  
Admin analyzes clinic statistics and activities  

3. Manage System  
Admin manages system settings  
Admin can control application configuration  

 Extended Scenarios

- Unauthorized access → Access denied message  
- Data error → System shows error message  

 Use Case

 Admin Dashboard

Actor: Admin  

Precondition:  
Admin must be logged in to the system.

Main Flow:  
1. Admin logs into the system  
2. Admin opens the dashboard  
3. Admin manages users or views reports  
4. System saves the changes  

Postcondition:  
System data is updated successfully.
